Dual Digital Twin

When you build the Digital Twin, OntoBricks creates two complementary representations:

  • Triple-Store — a SQL VIEW in Unity Catalog that maps directly to your source data with no duplication. Used for Quality checks.
  • Graph Database — a Graph DB embedded graph for Knowledge Graph exploration, GraphQL queries, and API access.
The Triple-Store VIEW is always created in the registry's catalog.schema. Its name is derived from the domain name and version (triplestore_<domain>_V<version>).
Lakebase (Postgres)

Graph triples are stored in the App-bound Lakebase Postgres instance. The logical table name follows your domain and version ({{ graph_name|default('', true) }}). Schema and database are configured under Settings → Graph DB.

Postgres is the source of truth for the graph store.
